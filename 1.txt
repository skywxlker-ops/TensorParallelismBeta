Performance
Fuse the two LayerNorms in the parallel block — gpt2_tp_test.cpp:376-377 computes ln1 and ln2 on the same input x sequentially. A single fused kernel reading x once and writing both outputs would halve memory bandwidth.

Avoid redundant contiguous() calls — gpt2_tp_test.cpp:122-124 calls .contiguous() on q, k, v after reshape+transpose. If your matmul already handles strided tensors, these are unnecessary copies.

Pre-allocate scale_t once — gpt2_tp_test.cpp:129-131 creates a new Tensor::full every forward call. Make it a cached member like cached_pos_.

Batch the replicated grad AllReduce — gpt2_tp_test.cpp:668-679 does one AllReduce per parameter. Pack all replicated grads into a single flat buffer and do one AllReduce.

Remove output.wait() in DEmbedding — CustomDNN.cpp:371 is a blocking sync in every embedding forward. Replicated embeddings don't need this.

Vocab-parallel cross entropy has 3 blocking AllReduces — CustomDNN.cpp:656-681. The sum-exp and target-logit AllReduces are independent and could be fused into one AllReduce on a [2*BT] buffer.

Memory
static variables in hot paths are risky — gpt2_tp_test.cpp:604 static Tensor grad_scale and CustomDNN.cpp:34-35 static float* d_layer_norms won't work correctly if you ever run multiple models or change devices.

DLinear allocates output DTensor every forward — CustomDNN.cpp:165-166 constructs a new DTensor with string name each call. Pre-allocate and reuse.

Correctness
DBlock::forward is broken — gpt2_tp_test.cpp:188-204 only does MLP residual, skipping attention entirely (comment says "placeholder"). Dead code since GPT bypasses it, but should be removed or fixed.

cudaGetDeviceCount called every embedding forward — CustomDNN.cpp:360-362. Cache the device count.
