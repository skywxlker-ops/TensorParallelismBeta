Algorithm Design - 

The flow of multi-GPU tensor operations this month focused on bringing DTensor sharding with our CustomDNN kernels, addressing Vocab Parallelism and Fused MLP operations.




System Design - 

The module is fully functional to ensure data flow between Custom C++ DNN, DTensor, Tensor-Implementations and NCCL backends is barrier-free.

MLP Implementations (Row / Column Parallel)

The feed-forward network splits the operations structurally - 

Layer 1 (Column-Parallel) - 

Matrix multiplication split across intermediate hidden dimensions. 

No internal synchronization is needed.

Layer 2 (Row-Parallel) -

Matrix multiplication across the output dimensions. 

I implemented a Fused DMLP Forward pass by adding a no_sync flag to DLinear::forward. 

The internal sub-syncs are skipped and unified into a single synchronization at the tail end of the entire MLP block, drastically increasing single-node and multi-node throughput.

DTensor and CustomDNN Integration

Fused MLP Execution - 

To prevent the GPU stream from blocking on intermediary synchronizations, DLinear::forward was adapted to accept a no_sync flag. 

The CustomDNN implementation of DMLP now routes column-parallel and row-parallel computations successively without waiting, before executing a unified fused AllReduce via the physical DTensor network call.

Vocab Parallel Processing - 

Processing shifted from full-tensor replication to vocab-dimension sharding. 

The input target dimensions are processed locally per rank. 

CustomDNN computes the maximum logits and exponential sums in local C++ kernels, after which DTensor invokes asynchronous networking to retrieve the global_max and global_sum_exp required for numerically stable block-distributed loss computations.





System Optimizations

GPU-Side Gradient Norm Calculation - 

Previously, compute_grad_norm() copied parameter gradients to the CPU to iterate via a slow loop.

I shifted this to a complete GPU-side operation using DTensor::grad_norm(), requiring only a single all_reduce_cpu of a floating point.

DataLoaderLite Device Allocation Fixes - 

 Implemented automatic device alignment using cudaSetDevice() to resolve "CPU->GPU transfer failed: invalid argument" errors in multi-GPU setup. 

Tensors are allocated dynamically matching the rank, adopting a CPU-first approach with .to(gpu_device).

Autograd Engine Fixes (NaN Avoidance) - 

Modified DTensor::reduce_grad to return an active view of the tensor rather than modifying the autograd state in-place. 

Because the engine was "stacking" recursive all-reduces on identical memory, early training ran precisely into NaN saturation boundaries.

Pseudocode / Program Logic (customDNN module)

Logic - 

Generalized Distributed Module Architecture

DTensor CustomLayer::forward(DTensor& input) {
   // 1. Setup local output layout matching the global mesh
   Layout out_layout(mesh, {batch, out_local});
   DTensor output(mesh, pg, out_layout, "layer_output");


   // 2. Local Math Dispatch (e.g., Matrix Multiplication)
   // For Column-Parallel: No cross-rank dependency exists in forward
   // For Row-Parallel: Intermediate results must be summed
   if (is_column_parallel) {
       output.linear_w_autograd(input, *weight, *bias);
       // Sync-free -  Results are naturally sharded correctly.
   }
   else if (is_row_parallel) {
       output.linear_w_autograd(input, *weight);
      
       // 3. Collective Communication (Sync)
       // Aggregates partial products across the mesh dimension
       if (!no_sync) {
           output.sync(); // All-Reduce Sum (Gradient identity in backward)
       }
      
       // Add bias AFTER sync to prevent multi-GPU duplication
       if (has_bias) {
           output.mutable_tensor() += bias->mutable_tensor();
       }
   }
  
   return output;
}



Core Logic -

Asynchronous Execution -  

Operations are dispatched to the GPU stream with no_sync flags whenever possible to overlap communication with the next layer's math.

Autograd Integration - 

CustomDNN manually binds gradients and ensures that collective operations (like AllReduce) don't double-count gradients during the backward pass (using .sync() for identity backprop).

Device Alignment - 

Every module automatically detects its rank and ensures local_v or local_hidden dimensions are sliced before kernel launching.

Usage - 

Fused MLP Block (Communication Hiding)

We hide communication by skipping internal syncs and unifying them at the block boundary.

// DMLP usage in the GPT-2 block
DTensor DMLP::forward(const DTensor& input) {
   // Column-parallel: naturally asynchronous (No sync required)
   DTensor h = fc1_->forward(input);
   DTensor h_act = h.gelu();
  
   // Row-parallel: Pass no_sync=true to evade internal stream blocking
   DTensor output = fc2_->forward(h_act, /*no_sync=*/true);
  
   // Unified sync: Triggers one single NCCL AllReduce for the entire MLP
   output.sync();
   return output;
}
