Phase-wise Optimization Plan for GPT-2 Tensor Parallelism
This plan addresses the performance, memory, and correctness issues identified in the performance review. It is structured into 4 phases, prioritizing "quick wins" with high impact and low effort.

Proposed Changes
Phase 1: Quick Wins & Correctness (High Impact, Trivial Effort)
Focusing on caching and removing unnecessary blocking syncs.

Remove output.wait() in DEmbedding forward: Remove blocking output.wait() in CustomDNN.cpp since replicated embeddings don't necessitate it.
Cache scale_t allocation: Pre-allocate scale_t as a class member in the attention forward function (gpt2_tp_test.cpp) rather than creating Tensor::full every call.
Cache cudaGetDeviceCount: Cache the device count result in DEmbedding::forward (CustomDNN.cpp) instead of calling it every time.
Phase 2: Communication Optimizations (High Impact, Low-Medium Effort)
Minimizing overhead from collective communication primitives.

Batch replicated grad AllReduce: At the end of the backward pass (gpt2_tp_test.cpp), pack all replicated parameter gradients into a single flat buffer and perform one AllReduce call.
Fuse Cross-Entropy AllReduces: In vocab-parallel CE (CustomDNN.cpp), fuse the independent sum-exp and target-logit AllReduces into a single AllReduce on a combined buffer.
Phase 3: Memory Allocations & Hot Path Safety (Medium Effort)
Focusing on reusing allocations and cleaning up unsafe static variables.

Reuse DLinear output buffer: Pre-allocate the output DTensor in the DLinear component (CustomDNN.cpp) and reuse it during subsequent forward passes.
Remove unsafe static variables: Remove static variables in hot paths (grad_scale in gpt2_tp_test.cpp and d_layer_norms in CustomDNN.cpp) to ensure correctness across multiple devices/models.
Phase 4: Kernels and Advanced Optimizations (Medium-High Effort)
Tackling specific kernel operations and dead code.

Remove redundant .contiguous() calls: Remove .contiguous() on q, k, v after reshape+transpose (gpt2_tp_test.cpp) if the underlying matmul implementation handles strided inputs correctly.
Fix or remove DBlock::forward: Fix or remove the broken DBlock::forward implementation (gpt2_tp_test.cpp), which currently skips attention.
Fused dual LayerNorm: Implement a custom kernel for the parallel block (gpt2_tp_test.cpp) that reads x once and writes both ln1 and ln2 outputs simultaneously, halving memory bandwidth.
Verification Plan
Automated Tests
Build and run existing GPT-2 tests (e.g., ./gpt2_tp_test) after each phase.
Verify that testing matches the baseline loss convergence to ensure numerical correctness.
Manual Verification
Manually review Nsight Systems (Nsys) profiles or internal timing logs to measure and confirm the expected performance gains in throughput.
